Check the latest versions below. Each release includes categorized changes to help you prioritize updates.

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time collaboration for shared docs with live cursors and comments.
- Introduced custom brand colors via YAML frontmatter, like `brandColor: "#3B82F6"`.
- Enabled Mermaid diagram support in all MDX pages for flowcharts and sequences.

## Improvements

- Enhanced search with fuzzy matching and syntax highlighting in results.
- Optimized page load times by `>30%` through lazy-loading components.

## Bug Fixes

- Fixed parsing errors with nested code blocks using 4 backticks.
- Resolved JSX attribute validation for arrays like `tags={["feature"]}`.

</Update>

## Migration Guide

Follow these steps to upgrade smoothly. Test in a staging environment first.

<Steps>
  <Step title="Backup Your Docs" icon="database-backup">
    Export all `.mdx` files to prevent data loss.
  </Step>
  <Step title="Update Package" icon="package">
    <CodeGroup tabs="npm,yarn">
      ```bash
      npm install comparetech-docs@latest
      ```
      ```bash
      yarn add comparetech-docs@latest
      ```
    </CodeGroup>
  </Step>
  <Step title="Review Changes" icon="file-diff">
    Scan for deprecated components and update frontmatter.
  </Step>
  <Step title="Test Parsing" icon="check-circle">
    Run `npm run docs:build` to validate MDX syntax.
  </Step>
</Steps>

<Expandable title="Detailed Migration Notes" default-open="false">

For v1.2.0, update code blocks:

````jsx
<CodeGroup tabs="JavaScript,Python">
  ```javascript
  // New async API
  const docs = await loadDocs({ theme: 'dark' });
  ```
  ```python
  # Python SDK update
  docs = load_docs(theme='dark')
  ```
</CodeGroup>
````

|---------|------------|-----------------|

| v1.1.0  | JSX attributes | Switch to double quotes |
